# 幣須知道！— 交接精華 (Handover Summary)

> 台灣政府補助與數位點數整合平台。核心價值:解決「點數分散」痛點,讓使用者一站看清所有可用補助/點數、資格、金額與到期日。
> 更新日期:2026-07-03

---

## 1. 技術棧與啟動

**技術棧:** Python 3 + Flask + SQLite｜前端單檔 SPA(Tailwind CDN + **Google 地圖(有金鑰)/ Leaflet(fallback)** + Chart.js)｜Google / Facebook / LINE OAuth + 訪客登入｜源泉圓體 webfont。

**啟動(第一次):**
```powershell
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env      # 填入金鑰(見第 5 節)
python init_db.py           # 建 DB：補助 15 筆、店家 27 筆
python app.py               # → http://localhost:5001
```
> ⚠️ 一律用 **localhost:5001**(Facebook 不接受 127.0.0.1)。改到補助資料才需重跑 `init_db.py`;只改畫面重新整理即可。

## 3. 資料模型 (SQLite)

**subsidy_list(補助主表,15 筆)** — 關鍵欄位:
- `category` 興趣類別:藝文/運動/旅遊/居住/就業/生活
- `use_type` **四大分類(用途)**:點數(A)/津貼(B)/場館(C)/節慶(D) ← 主要分類軸,跨縣市通用
- `target_city` 縣市(篩選維度)、`min_age`/`max_age`、`status`、`deadline`
- `value_min`/`value_max` **結構化金額(NT$)** ← 給精準試算,不再 parse 文字
- `age_flex` =1 表示有身分別例外(如身障/原民不限齡)→ 資格顯示「需確認」
- `amount`(顯示文字)、`description`/`how_to`/`notes`(懶人包)、`url`

**其他表:** `users`(google_id 為 PK,存各登入方式的 id/age/city/interest)、`stores`(消費地圖點位,27 筆,coin_type 分:藝文/運動/旅遊/運動卡/YouBike)、`user_claims`(券包)。

**資料現況:** 7 筆中央補助 + 5 筆桃園在地 + 3 筆桃園場館/節慶。多縣市擴充只需新增資料列(填 target_city + use_type + value),前端自動歸類,免改架構。

## 5. 金鑰與安全(交接必看)

- **金鑰全在 `.env`**(`.env.example` 為範本):`GOOGLE_CLIENT_ID/SECRET`、`FACEBOOK_CLIENT_ID/SECRET`、`LINE_CLIENT_ID/SECRET`、`FLASK_SECRET`、`GOOGLE_MAPS_API_KEY`。FB/LINE 沒填則該按鈕顯示「尚未設定」;**Maps 金鑰沒填則地圖自動退回 Leaflet+OSM(功能正常)**。
- **`GOOGLE_MAPS_API_KEY`(選填):** 到 Google Cloud Console 啟用「Maps JavaScript API」+「Directions API」建金鑰;有填→消費地圖/行程改用 Google 底圖 + 真實導航路線,沒填→退回 OSM。務必在金鑰設「HTTP 參照網址」限制(`http://localhost:5001/*` + 正式網域)防盜刷。
  - ✅ **已設定(2026-07-03):** 金鑰建於 Google Cloud 專案「My First Project」(名稱 `Maps Platform API Key`),已限制 HTTP 參照網址 `http://localhost:5001/*` + Maps API,已填入 `.env`。帳單:已開 90 天 $300 免費試用。**上正式站要在金鑰加正式網域的 referer**。
- 🔴 **舊 Google client secret 曾外流** → 上線前務必到 Google Cloud Console **重新產生**。
- **重新導向 URI**(三家都要註冊 localhost 版):
  - Google `http://localhost:5001/login/callback`
  - Facebook `http://localhost:5001/login/facebook/callback`
  - LINE `http://localhost:5001/login/line/callback`
- `app.run(debug=True)` 僅供開發,部署要關並改用正式 WSGI。
- `init_db.py` 每次執行會 DROP 全表(含 users)→ 正式環境要改成非破壞性。

## 6. 核心架構

- **後端(app.py):** 負責 OAuth 登入與供資料。`index()` 一次把 subsidies / stores / user / claimed 以 JSON 注入模板。
- **前端(index.html JS):** 負責所有渲染與邏輯(資格判斷 `eligStatus`、篩選、省錢試算、錢包、地圖)。
- **資料驅動:** 結構化欄位(value_*、age_flex、use_type)讓計算精準、可擴充。

> ⚠️ 架構債:核心運算(資格/試算)目前全在前端。要達金融級零容錯,建議未來抽到 Flask API + 單元測試(前後端邏輯分離)。
