# 新手啟動教學（保母級）· 補助快篩網

這份是給第一次跑這個專案的你。照著順序做就會成功。

---

## 一、先搞懂「哪個檔案是做什麼的」

| 檔案 / 資料夾 | 這是什麼 | 你需要開它嗎？ |
|---|---|---|
| **`app.py`** | 🟢 **網站總開關**。執行它,網站就跑起來 | 執行專案就是跑這支 |
| `templates/index.html` | 網站的主畫面(首頁、快篩、地圖、行程) | 想改畫面才開 |
| `templates/login.html` | 登入頁畫面 | 想改登入頁才開 |
| `init_db.py` | 建立資料庫、放補助/店家資料 | 想改補助或店家資料才開 |
| `subsidies.db` | 資料庫(由 `init_db.py` 自動產生) | ❌ 不用手動開 |
| `.env` | 放你的密鑰(要自己建立) | 設定時開一次 |
| `.env.example` | `.env` 的填寫範本 | 參考用 |
| `requirements.txt` | 需要安裝的套件清單 | ❌ 不用開,指令會用到 |
| `README.md` / `交接文件.md` | 說明文件 | 想看說明才開 |
| `__pycache__` | Python 自動產生的快取 | ❌ 不用理它 |

**一句話結論:要讓網站跑起來,就是執行 `app.py`。**

## 二、第一次啟動（完整步驟）

### 步驟 0：清掉殘留檔（如果有的話）

如果資料夾裡有 `subsidies.db-journal` 這個檔,代表上次建資料庫沒跑完。
在檔案總管裡把 **`subsidies.db`** 和 **`subsidies.db-journal`** 兩個一起選起來,按 `Delete` 刪掉。
（別擔心刪錯,步驟 4 會重新產生一份乾淨的。）

### 步驟 1：開啟 PowerShell

在 `project_new` 資料夾的**空白處**,按住鍵盤 `Shift` + 滑鼠右鍵 → 選「**在此開啟 PowerShell 視窗**」。
（看到一個黑色或藍色的文字視窗就對了。）

### 步驟 2：確認有裝 Python

貼上這行,按 Enter：
```powershell
python --version
```
- 有看到版本號(例如 `Python 3.12.0`)→ 很好,繼續。
- 出現錯誤或跳出微軟商店 → 你還沒裝 Python。去 https://www.python.org/downloads/ 下載安裝,**安裝時記得勾選「Add Python to PATH」**,裝完關掉 PowerShell 重開一次。

### 步驟 3：建立虛擬環境 venv 並安裝套件（只需第一次）

先建立一個專屬這個專案的 Python 環境(venv),再安裝套件：
```powershell
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
```
- 第二行成功後,**行首會出現 `(venv)` 字樣**,代表已進入虛擬環境。
- 最後看到 `Successfully installed ...` 就代表套件裝好了。

> 💡 venv 的好處:套件只裝在這個專案資料夾裡,不會跟你電腦其他 Python 專案打架。

### 步驟 4：設定你的密鑰

先把範本複製成 `.env`：
```powershell
copy .env.example .env
notepad .env
```
記事本會打開 `.env`,把裡面三行改成你自己的：
```
GOOGLE_CLIENT_ID=你的_client_id.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=你的_新_client_secret
FLASK_SECRET=隨便打一串英數字亂碼都可以
```
> ⚠️ Google 金鑰要去 **Google Cloud Console** 重新產生一組(舊的已外流不能用)。
> 改好後：檔案 → 儲存 → 關掉記事本。

### 步驟 5：建立資料庫

```powershell
python init_db.py
```
看到 `🎉 資料庫初始化成功！補助方案 7 筆、消費店家 17 筆。` 就成功。

### 步驟 6：啟動網站

```powershell
python app.py
```
看到 `Running on http://127.0.0.1:5001` 就成功了！

### 步驟 7：打開網站

瀏覽器網址列輸入：**http://127.0.0.1:5001** → 按 Enter。

**要關掉網站**：回到 PowerShell 視窗,按 `Ctrl + C`。

---

## 三、以後要再打開專案（第二次以後）

套件和 `.env` 都設定過了,不用重做。開啟 PowerShell 後,**先啟用 venv 再啟動**：
```powershell
.\venv\Scripts\activate
python app.py
```
然後瀏覽器開 http://127.0.0.1:5001。

> 每次重新開 PowerShell 都要先跑一次 `.\venv\Scripts\activate`（看到行首出現 `(venv)` 就對了),否則會出現「找不到 flask」的錯誤。

---

## 四、我只想改資料 / 改畫面，要重建嗎？

- **改補助或店家內容** → 編輯 `init_db.py` 裡的資料 → 重跑 `python init_db.py`（會自動覆蓋舊資料）→ 再 `python app.py`。
- **改畫面** → 編輯 `templates/index.html` → 重跑 `python app.py` 即可（資料庫不用動）。
- **套件不用再裝**、**`.env` 不用再設**,那些是一次性的環境設定。

## 五、常見錯誤排解

| 畫面出現 | 意思 | 怎麼辦 |
|---|---|---|
| `RuntimeError: 缺少 Google OAuth 金鑰` | 沒設定 `.env` 或金鑰是空的 | 回步驟 4,確認 `.env` 存在且填了金鑰 |
| `ModuleNotFoundError: No module named 'flask'` | 套件沒裝 | 回步驟 3 跑 `pip install -r requirements.txt` |
| `python 不是內部或外部命令` | 沒裝 Python 或沒加到 PATH | 回步驟 2 重裝並勾 Add to PATH |
| `no such table: subsidy_list` | 還沒建資料庫 | 跑步驟 5 `python init_db.py` |
| 網頁打不開 / 連線被拒 | 網站沒在跑 | 確認 PowerShell 有顯示 `Running on ...`,沒有就重跑步驟 6 |
| `Address already in use`（port 被占用） | 上一個還在跑 | 在跑著的那個視窗按 `Ctrl + C` 關掉,或關掉舊視窗重開 |

> 小提醒：網站在跑的時候,那個 PowerShell 視窗**不要關**,關了網站就停了。要一邊跑一邊打指令的話,再開一個新的 PowerShell 視窗。
