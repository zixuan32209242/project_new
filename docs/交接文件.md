# 補助快篩網 · 專案交接文件

> 台灣補助整合平台 — 依使用者的年齡、縣市、興趣，自動篩選並整理台灣各項補助，附懶人包、日曆提醒、消費地圖與一日遊行程規劃。

**交接日期:** 2026-07-01
**技術棧:** Python 3 + Flask + SQLite + Google OAuth（前端純 HTML/JS + TailwindCSS + Leaflet）

---

## 1. 一分鐘快速理解

這是一個「政府補助推薦平台」。使用者用 Google 登入 → 填年齡/縣市/興趣問卷 → 系統依條件篩選出符合資格的補助（文化幣、運動幣、租金補貼等），並提供懶人包說明、加入日曆提醒、消費地圖找店家、以及依已領券包自動排一日遊行程。

整個網站是**單頁應用（SPA）**:後端一次把所有資料送給前端,前端用 JavaScript 切換四個分頁,幾乎不再重新整理頁面。

## 2. 專案結構

```
project_new/
├── app.py               # Flask 後端（登入、篩選、領取、API）
├── init_db.py           # 一鍵建立資料庫 + 塞入示範資料
├── subsidies.db         # 執行 init_db.py 後自動產生（不在版控內）
├── requirements.txt     # 套件清單：Flask, Authlib, requests
├── README.md            # 原始說明文件
└── templates/
    ├── login.html       # Google 登入頁
    └── index.html       # 主程式（4 大分頁 SPA，約 900 行）
```

> ⚠️ 注意:程式碼 `app.py` 用 `render_template('login.html')` 與 `'index.html'`,代表兩個 html 需放在 **`templates/`** 資料夾下 Flask 才找得到。目前它們直接放在專案根目錄,啟動前請確認已移入 `templates/`。

## 3. 如何啟動（本機）

```powershell
# 1. 安裝套件
pip install -r requirements.txt

# 2. 建立資料庫（第一次，或改資料後執行）
python init_db.py

# 3. 啟動
python app.py
# 開瀏覽器 → http://127.0.0.1:5001
```

Google 登入需在 Google Cloud Console 把「授權的重新導向 URI」設為:
`http://127.0.0.1:5001/login/callback`

## 4. 後端 `app.py` 說明

啟動於 **port 5001**,`debug=True`。核心路由:

| 路由 | 方法 | 作用 |
|---|---|---|
| `/` | GET | 首頁。未登入回 `login.html`；已登入撈使用者狀態+所有補助+店家+已領清單,一次送給 `index.html` |
| `/login` | GET | 導向 Google OAuth |
| `/login/callback` | GET | Google 回呼,寫入 session 與 `users` 表 |
| `/logout` | GET | 清空 session |
| `/update_profile` | POST | 更新問卷資料（年齡/縣市/興趣），首次登入 onboarding 用 |
| `/claim` | POST | 領取/取消領取一張券（toggle），供行程規劃用 |
| `/api/subsidies` | GET | 純資料 API（回傳所有補助 JSON） |
| `/api/stores` | GET | 純資料 API（回傳所有店家 JSON） |

小工具函式:`get_db()` 回傳 row_factory 為 dict 的連線;`rows_to_dicts()` 把查詢結果轉成字典陣列。

## 5. 資料庫 `init_db.py` 說明

執行時**會先 DROP 掉舊表再重建**（開發階段強制重置,正式上線前要拿掉,否則每次跑都會清空使用者資料）。

四張表:

**`subsidy_list`（補助方案主表）**
欄位:`id, title, category(藝文/運動/旅遊/居住/就業), icon, min_age, max_age, target_city, amount, start_date, deadline, status, description(懶人包), how_to(怎麼領), notes(加碼/注意), url`

**`users`（使用者）**
`google_id(PK), email, name, picture, age, city, interest` — 後三欄由問卷更新。

**`stores`（消費店家,給地圖用）**
`id, name, coin_type(藝文/運動/旅遊), city, district, address, lat, lng, note`

**`user_claims`（領取紀錄,給行程規劃用）**
`id, google_id, subsidy_id, claimed_at`,`UNIQUE(google_id, subsidy_id)`

## 6. 前端 `index.html` 說明（SPA）

依賴 CDN:TailwindCSS、Font Awesome、Leaflet（地圖）。後端用 Jinja2 把資料注入,前端 JS 接手渲染。

**流程:** 若使用者尚未填 `age`/`city` → 顯示問卷 onboarding 卡片（`submitInlineSetup()` 打 `/update_profile`）→ 完成後進入主介面。

**四大分頁**（由 `navigate(page)` 切換,桌面左側導覽 + 手機底部導覽）:

1. **首頁 dashboard** (`renderDashboard`) — 「你可能會喜歡」依條件加權排序（`scoreForUser`）。
2. **快篩 search** (`renderSearch` / `runFilter`) — 依地區/興趣/年齡篩選,`eligible()` 判資格、`isActive()` 判是否開放。
3. **地圖 map** (`initMap` / `filterMap` / `locateMe`) — Leaflet 顯示店家,可按券別過濾,`locateMe()` 用 `navigator.geolocation` 定位找最近。
4. **行程 itinerary** (`renderItinerary`) — 依已領券包（user_claims）自動排一日遊,畫在 Leaflet 上。

**其他關鍵函式:** `openDetail()` 開券詳情彈窗、`toggleClaim()` 打 `/claim` 領/退券、`addToCalendar()`/`downloadICS()` 加 Google 日曆或下載 .ics、`haversine()` 算兩點距離。
